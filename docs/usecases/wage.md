# ユースケース: 賃金・割増率管理

割増賃金の計算に必要なマスタと設定を管理する。
社員の雇用形態によって単価・割増率の管理方法が異なる。

- **正社員・契約社員**: 職位マスタに基本時給と割増率を設定し、社員に職位を紐付ける。
- **派遣社員**: 所属する派遣元会社を紐付けたうえで、時給と割増率は社員個人に直接設定する。派遣元会社内でも職位等により個人差があるため、会社単位での一律設定は行わない。

---

## UC-WG01: 職位マスタの管理

- **アクター**: 管理者
- **概要**: 正社員・契約社員の職位ごとに基本時給と割増率を設定・管理する。
- **基本フロー**:
  1. 管理者が職位名（例: 一般職 / 主任 / 係長 / 管理職）を登録する。
  2. 各職位に以下を設定する:
     - 基本時給
     - 残業割増率（法定下限: 1.25、例: 1.30）
     - 深夜割増率（法定下限: 1.25、例: 1.25）
     - 休日出勤割増率（法定下限: 1.35、例: 1.35）
     - 深夜残業割増率（残業＋深夜の組み合わせ、法定下限: 1.50）
  3. 既存職位の名称・割増率を変更できる。
  4. 使用しなくなった職位を無効化する（過去データへの影響はなし）。
- **備考**: 管理職など労働基準法上の残業規定が適用されない職位は、割増率を0に設定できる。

---

## UC-WG02: 派遣元会社マスタの管理

- **アクター**: 管理者
- **概要**: 派遣社員の所属会社（派遣元）を登録・管理する。単価や割増率はここでは設定せず、社員個人に直接設定する（UC-WG04）。
- **基本フロー**:
  1. 管理者が派遣元会社名・連絡先などの基本情報を登録する。
  2. 既存会社の情報を編集できる。
  3. 取引終了となった会社を無効化する（所属社員がいる場合は警告を表示する）。

---

## UC-WG03: 割増判定ルールの設定

- **アクター**: 管理者
- **概要**: 残業・深夜・休日出勤それぞれの割増が適用される条件を定義する。
- **基本フロー**:
  1. 管理者が以下のルールを設定する:
     - **残業**: 社員に割り当てられたシフト（UC-SH02）の開始前・終了後の勤務時間を残業とする。シフト未設定日の全勤務時間も残業扱いとする。法定上限（1日8時間・週40時間）を超えた分はシフト内であっても残業割増の対象とする。
     - **休日出勤**: 会社カレンダー（UC-WG06）で休日と定義された日の勤務。
  2. 深夜割増の適用時間帯は **22:00〜翌5:00で固定**（労働基準法第37条により変更不可）。
- **備考**: 残業と深夜が重複する時間帯は各割増率を合算して適用する（例: 残業1.25 ＋ 深夜0.25 = 1.50倍）。

---

## UC-WG04: 社員への単価・割増率の設定

- **アクター**: 管理者
- **概要**: 各社員に対して原価計算に使用する時給と割増率を設定する。正社員は職位マスタから継承し、派遣社員は個人に直接設定する。
- **基本フロー**:
  1. 管理者が社員を選択する。
  2. 雇用形態（正社員 / 契約社員 / 派遣社員）を選択する。
  3. **正社員・契約社員の場合**: 職位マスタから職位を選択する。時給・割増率は職位の設定値を初期値として継承し、個人単位で上書き可能。
  4. **派遣社員の場合**: 派遣元会社マスタから所属会社を選択する。時給・割増率は個人ごとに直接入力する。
- **事後条件**: 設定した単価・割増率が工数コスト計算に反映される。

---

## UC-WG05: 割増賃金の確認

- **アクター**: 管理者
- **概要**: 指定期間の打刻データをもとに、社員ごとの残業・深夜・休日出勤時間と割増コストを確認する。
- **基本フロー**:
  1. 管理者が対象社員と期間を指定する。
  2. システムが打刻データと割増判定ルールをもとに以下を算出する:
     - 通常勤務時間・コスト
     - 残業時間・割増コスト
     - 深夜時間・割増コスト
     - 休日出勤時間・割増コスト
     - 合計人件費
  3. プロジェクト別の工数に割増コストを按分して原価へ反映する。

---

## UC-WG06: 会社カレンダーの管理

- **アクター**: 管理者
- **概要**: 年度ごとに休日・所定休日を設定する。打刻データと照合することで休日出勤の判定に使用する。
- **基本フロー**:
  1. 管理者が対象年度を選択する。
  2. 以下の種別で休日を登録する:
     - **法定休日**: 週1日の法定休日となる曜日（通常は日曜）を設定する。
     - **国民の祝日**: 年度の祝日を一括インポートする（手動追加も可）。
     - **会社指定休日**: 夏季休暇・年末年始・創立記念日など会社が定める休日を個別に登録する。
  3. 登録済みカレンダーを月別に一覧表示して確認・修正できる。
- **事後条件**: 設定した休日が割増判定ルール（UC-WG03）の休日出勤判定に反映される。
- **備考**: 前年度のカレンダーをコピーして祝日のみ更新する運用を想定する。
