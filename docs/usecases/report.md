# ユースケース: レポート・集計

## UC-R01: 月次勤怠レポートの出力

- **アクター**: 社員（自分のみ）、直属の上司（担当社員分）、システム管理者（全社員分）
- **概要**: 指定月の出退勤打刻をもとに、勤務日数・勤務時間・残業時間などをまとめたレポートを出力する。
- **基本フロー**:
  1. 対象社員と対象月を指定する。
  2. システムが打刻データを集計し、日別の勤務時間・月次合計を算出する。
  3. 画面表示またはファイル（CSV / PDF）でダウンロードできる。
- **例外フロー**:
  - 退勤打刻が存在しない日は未退勤として集計し、レポート上でフラグを立てる。

---

## UC-R02: 工数レポートの出力

- **アクター**: 社員（自分のみ）、直属の上司（担当社員分）、プロジェクト管理者
- **概要**: 指定期間の工数をプロジェクト・タスク・社員ごとに集計したレポートを出力する。
- **基本フロー**:
  1. 期間・集計軸（プロジェクト / タスク / 社員）を指定する。
  2. システムが工数を集計し、一覧を表示する。
  3. CSV形式でダウンロードできる。

---

## UC-R03: 打刻漏れアラートの確認

- **アクター**: 直属の上司
- **概要**: 出勤打刻のみで退勤打刻がない社員、または当日打刻がない社員を一覧で確認する。
- **基本フロー**:
  1. 直属の上司がアラート一覧を開く。
  2. システムが当日時点で打刻漏れ（退勤未打刻・未出勤）の社員を表示する。
  3. 直属の上司は該当社員に連絡するか、打刻修正を促す。

---

## UC-R04: 月次締め処理

- **アクター**: システム管理者
- **概要**: 対象月の打刻・工数・経費データを確定し、以降の編集・削除を不可にする。
- **基本フロー**:
  1. システム管理者が対象月を選択する。
  2. システムが以下の未処理項目を一覧表示する:
     - 未承認の打刻修正申請・残業申請・休暇申請
     - 退勤打刻が存在しない日（未退勤）
     - 工数が未登録の打刻セッション
  3. 未処理項目を確認・対処したうえで締め処理を実行する（未処理があっても強制実行は可能だが警告を表示する）。
  4. 対象月のデータが「締め済み」状態になり、社員・上司による編集が不可となる。
- **事後条件**: 締め済み期間の打刻・工数・経費が編集・削除不可になる（他UCの例外フローの「締め済み」判定に対応）。
- **備考**: 締め処理後に修正が必要な場合は UC-R05（月次締め解除）を実施する。

---

## UC-R05: 月次締め解除

- **アクター**: システム管理者
- **概要**: 締め済みの月次データを一時的に解除して編集可能にする。影響範囲が大きいため操作ログを記録し、関係者に通知する。
- **基本フロー**:
  1. システム管理者が対象月を選択し、解除理由を入力する。
  2. システムが影響範囲（対象社員数・工数エントリ数・経費エントリ数）を表示して確認を求める。
  3. 確認後、対象月のデータを編集可能状態に戻す。
  4. 解除操作のログ（日時・実施者・理由）を記録し、直属の上司・プロジェクト管理者に通知する。
- **事後条件**: 修正完了後、UC-R04 で再度締め処理を行う必要がある。
- **備考**: 締め解除中はレポート・原価計算の確定値が変動する可能性があるため、関係者に周知する。
