# ユースケース: 打刻

## UC-P01: 出勤打刻

- **アクター**: 社員
- **概要**: 社員が出勤時に打刻を行い、出勤時刻を記録する。
- **事前条件**: システムにログイン済みであること。
- **基本フロー**:
  1. 出勤ボタンを押す。
  2. システムが現在時刻を出勤打刻として記録する。
  3. 打刻完了メッセージを表示する。
- **例外フロー**:
  - すでに出勤打刻済みの場合、エラーを表示して二重打刻を防ぐ。
  - 前日の退勤打刻が存在しない場合、警告を表示したうえで出勤打刻は受け付ける。打刻修正申請（UC-P04）を促すメッセージを表示する。

---

## UC-P02: 退勤打刻

- **アクター**: 社員
- **概要**: 社員が退勤時に打刻を行い、退勤時刻を記録する。
- **事前条件**: システムにログイン済みであること。当日の出勤打刻が存在すること。
- **基本フロー**:
  1. 退勤ボタンを押す。
  2. システムが現在時刻を退勤打刻として記録する。
  3. 当日の勤務時間を表示する。
- **例外フロー**:
  - 出勤打刻がない状態で退勤打刻しようとした場合、エラーを表示する。

---

## UC-P03: 打刻履歴の確認

- **アクター**: 社員、直属の上司
- **概要**: 社員自身または直属の上司が打刻の履歴を一覧で確認する。
- **基本フロー**:
  1. 対象の社員・期間を指定する。
  2. 出勤・退勤の打刻一覧を表示する（日付・時刻・種別）。

---

## UC-P04: 打刻修正の申請

- **アクター**: 社員
- **概要**: 社員が打刻漏れや誤打刻に気づいた際に、修正を申請する。
- **基本フロー**:
  1. 社員が対象の打刻を選択する（または新規打刻を申請する）。
  2. 修正後の日時と修正理由を入力する。
  3. システムが修正申請を直属の上司へ通知する。
- **事後条件**: 申請が「承認待ち」状態で登録される。

---

## UC-P05: 打刻修正申請の承認・却下

- **アクター**: 直属の上司
- **概要**: 直属の上司が社員からの打刻修正申請を確認し、承認または却下する。
- **基本フロー**:
  1. 直属の上司が申請一覧を確認する。
  2. 申請内容（修正前後の時刻・理由）をレビューする。
  3. 承認する場合、打刻データを修正内容で更新する。
  4. 却下する場合、却下理由を入力して社員に通知する。
