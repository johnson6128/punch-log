# ロール定義

各ユースケースで登場するアクターの役割と権限範囲を定義する。

| ロール | 説明 |
|---|---|
| **社員** | 自身の打刻・工数登録・各種申請を行う一般ユーザー |
| **直属の上司** | 担当社員のシフト管理・残業/休暇/打刻修正などの申請承認・勤務状況の確認を行う |
| **プロジェクト管理者** | プロジェクトの登録・予算設定・原価/売上管理・収支確認を行う |
| **システム管理者** | 社員アカウント・各種マスタ（職位/派遣元会社/カレンダー/休暇種別など）・システム設定を管理する |

1人の人間が複数ロールを兼務することもある（例: プロジェクト管理者が直属の上司を兼ねるなど）。
ロールの割り当て・変更は UC-E05 で行う。

## 通知方式

各UCで「通知する」と記載されている箇所の通知手段（メール / アプリ内通知 / プッシュ通知）は設計フェーズで確定する。最低限アプリ内通知を実装し、メール通知はオプションとして検討する。
