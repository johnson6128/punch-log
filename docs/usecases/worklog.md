# ユースケース: 工数

## UC-W01: 工数の登録

- **アクター**: 社員
- **概要**: 社員が打刻セッションに対して作業内容・時間を工数として登録する。
- **基本フロー**:
  1. 社員が退勤打刻後、または工数一覧画面から対象の打刻セッション（出退勤ペア）を選択する。
  2. プロジェクトをリストから選択する。
  3. タスクカテゴリ（管理者が定義したマスタ）をリストから選択する。
  4. 作業時間・作業内容（任意）を入力して保存する。
  5. システムが工数を保存し、当該セッションの合計工数を更新する。
- **例外フロー**:
  - 入力した工数の合計がセッションの勤務時間を超える場合、警告を表示する。
  - クローズ済みプロジェクトは選択できない。

---

## UC-W02: 工数の編集

- **アクター**: 社員
- **概要**: 登録済みの工数エントリを修正する。
- **事前条件**: 対象の工数が登録済みであること。
- **基本フロー**:
  1. 社員が工数一覧から対象エントリを選択する。
  2. 内容を修正して保存する。
- **例外フロー**:
  - 締め済みの期間に属する工数は編集不可とし、エラーを表示する。

---

## UC-W03: 工数の削除

- **アクター**: 社員
- **概要**: 誤って登録した工数エントリを削除する。
- **基本フロー**:
  1. 社員が工数一覧から対象エントリを選択する。
  2. 削除を確認後、システムがエントリを削除する。
- **例外フロー**:
  - 締め済みの期間に属する工数は削除不可とし、エラーを表示する。

---

## UC-W04: 工数一覧の確認

- **アクター**: 社員、直属の上司、プロジェクト管理者
- **概要**: 期間・プロジェクト・タスクなどの条件で工数エントリを一覧表示する。
- **基本フロー**:
  1. 対象の社員・期間・プロジェクトを指定する（任意）。
  2. 条件に合致する工数エントリを一覧表示する。

---

## UC-W05: 工数の集計

- **アクター**: 社員、直属の上司、プロジェクト管理者
- **概要**: プロジェクト・タスク・社員・期間ごとに工数を集計して確認する。
- **基本フロー**:
  1. 集計軸（プロジェクト / タスク / 社員）と期間を指定する。
  2. システムが合計工数を算出して表示する。
