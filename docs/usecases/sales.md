# ユースケース: 営業・稟議管理

## UC-SL01: プロジェクト提案の作成

- **アクター**: 営業
- **概要**: 営業担当者が新規プロジェクトの提案を作成し、稟議前の準備段階として登録する。
- **基本フロー**:
  1. 営業がプロジェクト提案画面を開く。
  2. 以下の情報を入力して保存する:
     - プロジェクト名・概要
     - 想定期間（開始予定日・終了予定日）
     - 想定売上目標
     - 担当営業
     - 備考（顧客情報・背景など）
  3. システムがステータス「提案中」でプロジェクトを登録し、**連携ID**（稟議システムとのリンク用の一意なID）を自動発行する。
- **例外フロー**:
  - アクティブなプロジェクト（提案中・稟議中・実行中）と同一名称の場合、警告を表示するが登録は許容する。
- **事後条件**: 提案中ステータスのプロジェクトが作成される。予算の執行（工数・経費の登録）は稟議承認後まで不可。
- **備考**: この段階では予算の確定・担当社員の割り当ては任意入力。詳細な予算設定はプロジェクト管理者が稟議承認後に行う（UC-PJ01）。

## UC-SL02: 稟議連携情報の登録

- **アクター**: 営業
- **概要**: 外部の稟議システムに提出した稟議の番号（稟議No.）をプロジェクトに記録し、ステータスを「稟議中」に移行する。
- **事前条件**: 対象プロジェクトが「提案中」ステータスであること。
- **基本フロー**:
  1. 営業が対象プロジェクトを選択する。
  2. 稟議No.（外部稟議システムで採番された番号）を入力する。
  3. 必要に応じて稟議提出日・稟議内容の概要を入力する。
  4. 「稟議申請」操作を実行すると、ステータスが「稟議中」に更新される。
- **例外フロー**:
  - 「提案中」以外のステータスのプロジェクトに対してこの操作を行おうとした場合、エラーを表示する。
- **事後条件**: プロジェクトに稟議No.と連携IDが記録され、ステータスが「稟議中」になる。
- **備考**: 連携IDはシステム側が自動発行（UC-SL01）し、外部稟議システムへの参照情報として使用する。稟議No.は外部システム側の採番であり、本システムでは記録・表示のみ行う。

## UC-SL03: 稟議承認に伴うプロジェクト実行開始

- **アクター**: 営業、プロジェクト管理者
- **概要**: 外部稟議システムで承認が下りたことを本システムに反映し、プロジェクトを「実行中」ステータスに移行して予算執行を可能にする。
- **事前条件**: 対象プロジェクトが「稟議中」ステータスであること。
- **基本フロー**:
  1. 営業が対象プロジェクトを選択し、「稟議承認済み」操作を実行する。
  2. 稟議承認日を入力する。
  3. システムがステータスを「実行中」に更新する。
  4. プロジェクト管理者が予算・担当社員などの詳細設定を行う（UC-PJ01）。
- **例外フロー**:
  - 「稟議中」以外のステータスのプロジェクトに対してこの操作を行おうとした場合、エラーを表示する。
- **事後条件**: ステータスが「実行中」となり、工数・経費の登録（予算執行）が可能になる。
- **備考**: 稟議が否決された場合はステータスを「提案中」に戻し、再提案または取り下げができる。取り下げの場合は別途クローズ操作を行う。

## UC-SL04: 営業案件の分割・管理

- **アクター**: 営業
- **概要**: 長期プロジェクトを工程・フェーズ・クォーターなどの単位で複数の営業案件に分割し、それぞれに売上目標を設定して管理する。
- **事前条件**: 対象プロジェクトが「実行中」ステータスであること。
- **基本フロー**:
  1. 営業が対象プロジェクトを選択し、「営業案件の追加」を操作する。
  2. 以下の情報を入力して保存する:
     - 案件名（例: フェーズ1 / 2026年Q1 / 初期導入）
     - 対象期間（開始日・終了日）
     - 売上目標金額
     - 備考
  3. システムが案件をプロジェクトに紐付けて保存する。
  4. プロジェクト全体の売上目標が各案件の合計として更新される。
- **例外フロー**:
  - 案件の期間がプロジェクトの期間外の場合、警告を表示する（保存は許容する）。
- **事後条件**: プロジェクトに1件以上の営業案件が紐付けられ、案件単位での売上登録が可能になる。
- **備考**: 案件を分割しない場合、プロジェクト全体を1つの案件として扱う（暗黙の一括案件）。既存案件の期間・目標金額は後から変更可能。

## UC-SL05: 営業案件ごとの売上登録

- **アクター**: 営業
- **概要**: 実績売上（請求額・入金額）を営業案件単位で登録する。
- **事前条件**: 対象プロジェクトに営業案件が登録されていること。
- **基本フロー**:
  1. 営業が対象プロジェクト・営業案件を選択する。
  2. 以下を入力して保存する:
     - 売上金額
     - 計上日
     - 摘要（請求書番号・入金区分など）
  3. 複数回に分けて登録可能（分割請求・マイルストーン請求に対応）。
  4. システムが案件の売上合計・プロジェクト全体の売上合計を更新する。
- **例外フロー**:
  - クローズ済みプロジェクトには売上を登録できない。
- **事後条件**: 登録した売上がプロジェクト収支レポート（UC-PJ09）に反映される。
- **備考**: 案件を分割していない場合はプロジェクト直下に売上を登録する（UC-PJ07 と同等の操作）。
